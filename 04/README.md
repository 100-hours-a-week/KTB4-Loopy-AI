## 위클리 챌린지 4주차 회고

### 무엇을 했나

가상 데이터셋을 직접 만들어 가며 분류의 기본기를 한 바퀴 돌았다. 데이터 분할부터 시작해 K-NN, 네 가지 고전 분류기 비교, 데이터 증강, 활성화 함수, MLP, CNN까지 — sklearn으로 빠르게 감을 잡고, 신경망 두 개(MLP·CNN)는 PyTorch로 직접 층을 쌓아 학습 루프를 손으로 돌려봤다.

### 배운 것 (Keep)

가장 크게 남은 건 **"전처리가 모델만큼 중요하다"**는 걸 숫자로 본 경험이다. K-NN은 거리 기반이라 스케일링을 빼면 성능이 흔들리는데, `StandardScaler`를 학습 데이터로만 `fit`하고 검증·테스트엔 `transform`만 적용하는 누수 방지 패턴을 몸에 익혔다. 통계 전공으로 표준화 개념은 알았지만, "왜 test에 fit하면 안 되는가"를 코드 구조로 체화한 게 달랐다.

분류기 비교(3번)에서는 알고리즘마다 성격이 뚜렷하다는 걸 봤다. 같은 데이터에서 Perceptron이 0.76에 그치는 동안 SVM(RBF)은 0.94까지 갔다. 선형 분리가 안 되는 데이터에선 커널·앙상블 계열이 강하다는 교과서 명제를 직접 확인하니 머리에 박혔다.

### 어려웠던 것 (Problem)

**데이터 증강(4번)이 처음엔 오히려 성능을 떨어뜨렸다.** 학습 데이터를 150개 쓰고 증강했더니 정확도가 0.931에서 0.919로 내려갔다. 원인을 따져보니 이미 데이터가 충분해서 증강이 노이즈로만 작용한 것이었다. 학습셋을 60개로 줄이자 그제야 0.789 → 0.807로 증강 효과가 드러났다. **"증강은 만능이 아니라, 데이터가 부족할 때 일반화를 돕는 도구"**라는 걸 실패를 통해 배운 게 가장 값졌다.

CNN(7번)도 처음엔 full-batch로 30 epoch만 돌려 0.62에 머물렀다. 미니배치(`DataLoader`) + epoch 확대 + Dropout을 넣고서야 0.969까지 올랐다. 학습 루프를 직접 짜다 보니 배치 구성과 정규화가 결과를 얼마나 바꾸는지 체감했다.

### 다음에 해볼 것 (Try)

- **재현성**: 시드를 고정해도 토치 결과가 미세하게 흔들리는 부분이 있어, 여러 시드 평균으로 성능을 보고하는 습관을 들이고 싶다.
- **확장**: 7번 CNN을 8×8 digits 대신 MNIST로 키워 데이터 규모에 따른 차이를 보고, 4번 증강도 회전·이동 외에 노이즈·컷아웃 같은 기법을 비교해보고 싶다.
- **퀀트 연결**: 이번에 익힌 train/val/test 누수 방지·스케일링·과적합 관리는 WorldQuant 알파 검증(IS/OS 분리)과 그대로 통한다. 금융 시계열은 시간 누수(look-ahead)가 더 치명적이라, 같은 원칙을 시계열 분할(walk-forward)로 옮겨 적용해보는 게 다음 과제다.

### 한 줄 정리

알고리즘을 "돌리는 법"보다 **언제 무엇이 왜 효과가 있는지**를 작은 실험으로 확인한 게 핵심이었다. 특히 증강과 CNN에서 한 번씩 실패하고 고친 과정이 가장 많이 남았다.
