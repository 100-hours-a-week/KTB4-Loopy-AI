# [과제 1] 아웃풋 중심 망각곡선 복습 플래너 CLI 프로그램 제작
- 오늘 공부한 내용을 바탕으로 1일 뒤, 3일 뒤, 일주일 뒤, 한달 뒤에 자동으로 백지 복습(인출) 일정을 잡아주는 '아웃풋 중심 망각곡선 복습 플래너' CLI 프로그램 입니다.


## 📖 How to use (사용 방법)

**1. 오늘 공부한 주제 추가하기:**

   오늘 새로 학습한 핵심 키워드나 주제를 입력합니다.
   입력 즉시 파이썬의 datetime 라이브러리가 에빙하우스 망각곡선 이론에 기반한 4단계 복습일(1일, 3일, 7일, 30일 뒤)을 자동으로 계산하여 저장합니다.

**2. 오늘 꼭 복습해야 할 주제 확인:**

   현재 날짜를 기준으로 오늘 복습 예정일에 도달한 학습 주제들을 필터링하여 리스트로 보여줍니다.
   출력된 리스트를 보고 책을 덮은 상태에서 키워드만으로 내용을 떠올리는 '백지 복습(인출 연습)'을 수행합니다.

**3. 전체 학습 기록 및 복습 일정 보기:**

   그동안 저장된 모든 학습 주제와 각각의 4단계 복습 일정을 한눈에 정렬하여 확인합니다. 전체적인 복습 로드맵을 파악할 때 유용합니다.

**4. 프로그램 종료:**

   안내 메시지와 함께 안전하게 프로그램을 종료하고 루프를 빠져나옵니다.


## 📝 회고 (Review)

**1. Keep (만족스러웠던 점 및 유지할 점)**

   **실용적인 주제 선정:**
   단순한 관리 프로그램이나 계산기 대신, 현재 부트캠프를 들으며 나에게 가장 필요한 '학습 복습 타이머'를 주제로 선정하여 개발 몰입도가 높았습니다. 나중에 AI 모델이나 DB를 붙여 확장하기 좋은 씨앗 프로젝트를 만들었다는 점에서 만족스럽습니다.
   
   **구조화된 데이터 관리:**
   데이터 구조를 리스트 내 딕셔너리 구조(List of Dictionaries)로 설계하여, 질문이나 학습 기록이 누적되어도 유연하게 확장하고 관리할 수 있는 가독성 높은 코드를 작성했습니다.
   
   **datetime 내장 라이브러리 활용:**
   문자열로 날짜를 다루지 않고 파이썬 표준 라이브러리를 활용해 날짜 연산(timedelta)을 정확하게 구현해 냄으로써 실무 데이터 전처리에 필요한 기초 역량을 다질 수 있었습니다.

**2. Problem (어려웠던 점 및 아쉬운 점)**

   **데이터 휘발성 (휘발성 메모리):**
   현재 코드는 프로그램이 종료되면 입력했던 복습 일정 데이터가 모두 사라지는 메모리 저장 방식입니다. 매일 연속성 있게 사용해야 하는 플래너의 특성상, 데이터를 보존하지 못한다는 점이 가장 아쉬웠습니다.
   
   **사용자 입력 예외 처리의 한계:**
   메뉴 선택(1~4)에 대한 예외 처리는 구현했으나, 나중에 날짜 지정 입력 등의 기능을 추가할 때 발생할 수 있는 자료형 에러(ValueError) 등에 대한 정교한 방어 코드가 아직 부족합니다.

**3. Try (해결 방안 및 개선 방향)**

   **파일 입출력(I/O) 또는 데이터베이스 도입:**
   다음 단계에서는 파이썬의 json 모듈이나 sqlite3를 활용하여, 프로그램을 껐다 켜도 이전에 기록한 복습 일정이 텍스트 파일이나 DB에 그대로 저장되고 로드될 수 있도록 데이터 영속성을 구현해 볼 예정입니다.
   
   **복습 완료 체크 기능 추가:**
   단순히 일정을 보여주는 것을 넘어, 사용자가 해당 주제의 백지 복습을 완료했는지 여부(True/False)를 체크하고 성공율을 통계로 보여주는 기능을 추가해 보고 싶습니다.
